# Push to Terminal Integration Guide

Integration with the Halo backend to send payment requests directly to a registered device via push notification.

## 1. Architecture

The Push to Terminal flow allows a merchant system to push a payment request to a specific device without the customer needing to scan a QR code. The process is as follows:

1. Merchant system calls `GET /devices` to retrieve the list of linked devices.
2. Merchant system selects the target device and calls `POST /push/terminal` with the payment details.
3. The Halo backend sends a Firebase push notification to the selected device.
4. The Halo.Go app on the device receives the notification, displays the payment request, and initiates a card read.
5. After the customer taps their card, the transaction result is sent to the merchant's callback URL via webhook.

If the push notification fails (device offline, invalid token), the backend automatically returns a fallback applink URL with a QR code that the customer can scan instead.

## 2. Device Management API

Retrieve all devices linked to the authenticated merchant.

<mark style={{color:'green'}}>`GET`</mark> `https://kernelserver.{env}.haloplus.io/devices`

## 3. Push to Terminal API

### 3.1 Send Payment Request

Push a payment request to a specific device via Firebase push notification.

<mark style={{color:'blue'}}>`POST`</mark> `https://kernelserver.{env}.haloplus.io/push/terminal`

<span/>

**Path Parameters**

| Name | Type | Description |
| ---- | ---- | ----------- |
| env<mark style={{color:'red'}}>\*</mark> | String | The backend environment \[dev, qa, prod] |

**Headers**

| Name | Type | Description |
| ---- | ---- | ----------- |
| Content-Type<mark style={{color:'red'}}>\*</mark> | String | Content Type of the Request: application/json |
| x-api-key<mark style={{color:'red'}}>\*</mark> | String | The API Key retrieved from the Merchant Portal (JWT can be used instead) |
| Authorization<mark style={{color:'red'}}>\*</mark> | String | The JWT retrieved from logging in. (API Key can be used instead) |

**Request Body**

| Name | Type | Description |
| ---- | ---- | ----------- |
| deviceInstallationId<mark style={{color:'red'}}>\*</mark> | String | The target device ID from `GET /devices` |
| amount<mark style={{color:'red'}}>\*</mark> | Number | Payment amount (minimum 0.01) |
| currency<mark style={{color:'red'}}>\*</mark> | String | 3-letter ISO currency code (e.g. ZAR, USD) |
| merchantReference<mark style={{color:'red'}}>\*</mark> | String | Unique reference for this payment |
| merchantId<mark style={{color:'red'}}>\*</mark> | String | Your Merchant ID from Merchant Portal |
| jwt<mark style={{color:'red'}}>\*</mark> | String | Configuration JWT for the payment session |

### 3.2 Webhook Callback

After a transaction completes on the device, send the result to the merchant's callback URL.

<mark style={{color:'blue'}}>`POST`</mark> `https://kernelserver.{env}.haloplus.io/push/terminal/webhook`

<span/>

**Path Parameters**

| Name | Type | Description |
| ---- | ---- | ----------- |
| env<mark style={{color:'red'}}>\*</mark> | String | The backend environment \[dev, qa, prod] |

**Headers**

| Name | Type | Description |
| ---- | ---- | ----------- |
| Content-Type<mark style={{color:'red'}}>\*</mark> | String | Content Type of the Request: application/json |
| x-api-key<mark style={{color:'red'}}>\*</mark> | String | The API Key retrieved from the Merchant Portal (JWT can be used instead) |
| Authorization<mark style={{color:'red'}}>\*</mark> | String | The JWT retrieved from logging in. (API Key can be used instead) |

**Request Body**

| Name | Type | Description |
| ---- | ---- | ----------- |
| transactionId<mark style={{color:'red'}}>\*</mark> | String | Transaction ID from the push response |
| merchantId<mark style={{color:'red'}}>\*</mark> | String | Your Merchant ID |
| status<mark style={{color:'red'}}>\*</mark> | String | Transaction result: `approved`, `declined`, `cancelled`, or `timeout` |
| amount<mark style={{color:'red'}}>\*</mark> | Number | Transaction amount |
| currency<mark style={{color:'red'}}>\*</mark> | String | 3-letter ISO currency code |
| merchantReference<mark style={{color:'red'}}>\*</mark> | String | Your payment reference |

## 4. Integration Example

<Tabs>
  <TabItem value="node" label="Node.js">

```javascript
const axios = require('axios');

const BASE_URL = 'https://kernelserver.prod.haloplus.io';
const API_KEY = 'your-api-key';

// Step 1: List devices
const devices = await axios.get(`${BASE_URL}/devices`, {
  headers: { 'x-api-key': API_KEY }
});

// Step 2: Pick a device and push payment
const device = devices.data.find(d => d.friendlyName === 'Front Counter');

const push = await axios.post(`${BASE_URL}/push/terminal`, {
  deviceInstallationId: device.deviceId,
  amount: 150.00,
  currency: 'ZAR',
  merchantReference: 'ORDER-123',
  merchantId: 'your-merchant-id',
  jwt: 'your-config-jwt'
}, {
  headers: {
    'x-api-key': API_KEY,
    'Content-Type': 'application/json'
  }
});

if (push.data.status === 'sent') {
  console.log('Push sent, transaction:', push.data.transactionId);
} else {
  console.log('Fallback -- display QR code to customer');
}
```
